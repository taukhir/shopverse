Backend Engineering Learning Path

Use this path to study backend engineering from fundamentals to production-oriented distributed-system behavior. Generic concepts come first; the corresponding Shopverse pages provide concrete implementation examples.

Complete each stage in order when learning the ecosystem for the first time. For interview revision or production troubleshooting, jump directly to the relevant stage and use the practical-outcome column as a checklist.

01Java and foundationsLanguage, OOP, collections, streams, concurrency, design principles, and REST.02Spring ecosystemBoot internals, web APIs, validation, dependency injection, and testing.03Data and persistenceRelational design, Hibernate, JPA, Liquibase, queries, and transactions.04Service communicationMicroservices, gateways, discovery, load balancing, Feign, and Kafka.05Distributed consistencyCAP, SAGA, outbox, CQRS, locks, resilience, and high availability.06SecuritySpring Security, JWT, OAuth2, RBAC, method security, and ownership.07Observability and deliveryLogs, metrics, traces, Docker, CI/CD, deployment, and troubleshooting.

Recommended Practical Sequence​

  1. Start the Docker stack.
  2. Log in and decode the JWT.
  3. Submit an idempotent checkout.
  4. Query the order timeline.
  5. Follow the correlation ID in Loki.
  6. Open the related trace in Zipkin.
  7. inspect request and business metrics in Prometheus or Grafana.
  8. Trigger a payment failure and observe compensation.
  9. Inspect an Outbox row and DLT recovery path.
  10. Run Quick, Integration, and Full verification modes.

After completing the generic path, use the Shopverse case study to connect the subjects in one working system. Its complete commands and expected outcomes are in Features and demonstrations.
