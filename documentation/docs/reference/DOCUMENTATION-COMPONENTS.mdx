# Reusable Documentation Components

These components are globally available in every Markdown and MDX page. No
import statement is required.

## Consistent Callouts

<DocCallout type="mistake">Do not describe planned behavior as currently implemented.</DocCallout>
<DocCallout type="production">Include failure behavior, observability, and recovery—not only the happy path.</DocCallout>
<DocCallout type="shopverse">Link implementation claims to source, configuration, or a runnable demonstration.</DocCallout>

Supported types are `mistake`, `production`, `shopverse`, `tip`, and `code`.

## Dependency And Command Tabs

`DependencyTabs` accepts `maven` and `gradle` content. `CommandTabs` accepts
`powershell` and `bash` content. The selected build tool or shell is remembered
across tab groups and can be represented in the page URL.

````mdx
<DependencyTabs
  maven={<>```xml
  &lt;dependency&gt;...&lt;/dependency&gt;
  ```</>}
  gradle={<>```kotlin
  implementation("group:artifact:version")
  ```</>}
/>

<CommandTabs
  powershell={<>```powershell
  .\\gradlew.bat test
  ```</>}
  bash={<>```bash
  ./gradlew test
  ```</>}
/>
````

`VersionTabs` accepts an `items` array with `label`, `value`, and `content`
when a guide must compare Java or framework versions.
